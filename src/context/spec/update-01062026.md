# Round 2 — Editorial Redesign (01/06/2026)

A full visual overhaul taking inspiration from https://www.bedanthota.com/ — a
minimal, dark, editorial portfolio with numbered sections — while keeping the
content and pushing the execution further. The old rainbow-gradient /
glassmorphism look was replaced with a quieter, more sophisticated system.

## Design system

- **Theme**: near-black base (`#08080a`), off-white text, a single restrained
  **violet accent** (`#8b5cf6`) used sparingly (numbers, links, hovers, icons)
  instead of the previous multi-colour soup.
- **Backdrop**: replaced the animated rainbow blobs with a fixed layered backdrop
  — a soft violet bloom + a masked fine dot-grid (`body::before` / `::after`).
- **Tokens**: added `--color-accent`, `--color-accent-soft`, `--color-muted`,
  `--color-line` to the Tailwind v4 `@theme` so `text-accent`, `border-line`, etc.
  are usable everywhere.
- **Reusable bits** in `globals.css`: `.eyebrow` (mono caps label), `.surface`
  (card), `.link-underline`, `.headline-tint`, `.hairline`, retuned
  `.gradient-text`, plus `pulse-dot` / `scroll-cue` animations. All heavy motion
  still disabled under `prefers-reduced-motion`.

## Structure (numbered sections, 01–06)

- **Hero**: typographic / editorial — availability pill, mono eyebrow, large
  gradient name, bold value-prop line, supporting copy, CTAs + social row, a
  mono meta strip, and an animated scroll cue. No more floating avatar.
- **01 About** (`ExpandableDescription`): two-column — bio + a 4-up **stats grid**
  on the left, and the **real photo** (`public/images/AP-1.jpg`) on the right as a
  framed, duotone-on-hover portrait with an accent offset frame. Read-more expands.
- **02 Experience**: editorial timeline with a vertical rail + accent nodes.
- **03 Projects**: full-width **featured Ghost AI** card (accent glow) above a
  2-col grid of the rest; year labels, mono tag chips, underline links.
- **04 Skills**: single bordered grid (hairline cell separators), mono category
  labels, individual tech chips.
- **05 Beyond the Code** (`Achievements.tsx`, now exported as `Beyond`): merged
  **Education + Achievements** into one two-column section.
- **06 What's Next** (`Contact.tsx`, new): large CTA headline, "Get in touch",
  and a 3-up contact tile row.

# Round 3 — Reveal hardening, centered hero, per-section components (01/06/2026)

Follow-up fixing a report that the Experience / Projects / Skills sections
could render blank, plus two structural requests.

## Reveal hardening (visibility fix)

- Root cause: sections relied on a single `whileInView` on the parent
  `<motion.section>` whose "animate" variant propagated down through nested
  `motion` containers to the leaf `fadeInUp` children. That one-shot reveal can
  be skipped on fast scrolls, smooth-scroll anchor jumps, or in Safari, leaving
  a whole section stuck at `opacity:0`. (About stayed visible only because its
  bio text was plain HTML, not motion-wrapped — which is why it never blanked.)
- New `src/components/Reveal.tsx`: a self-contained scroll-reveal wrapper. Each
  instance owns its own IntersectionObserver via `whileInView` with a forgiving
  viewport (`amount: 0.1`, `margin: "0px 0px -8% 0px"`, `once`) and an optional
  stagger `delay`. Content can no longer get stuck hidden.
- Refactored `SectionHeader`, `ExpandableDescription` (About), `WorkExperience`,
  `Projects`, `Achievements` (Beyond), and `Contact` to use `Reveal` instead of
  parent→child variant propagation. Sections are now plain `<section>` elements.
- Trimmed `lib/animations.ts` to just `fadeInUp` + `staggerContainer` (now used
  only by the hero's load-in); removed the unused `fadeIn` and `sectionInView`.

## Centered hero + per-section components

- Hero extracted into `src/components/Hero.tsx` and **centered** — content is now
  vertically and horizontally centered (`flex-col items-center justify-center
  text-center`) with centered CTAs and meta strip.
- Skills extracted into `src/components/Skills.tsx` (was inline in `page.tsx`).
- `page.tsx` is now a thin composition: `Navbar → Hero → About → Experience →
  Projects → Skills → Beyond → Contact → Footer`. Every section is its own
  component.

- `npx tsc --noEmit` — clean. `npx eslint src/` — clean.
- `npx jest` — 3/3 pass. `npm run build` — production build succeeds.
- Re-verified via Playwright/Chrome, including the previous failure mode
